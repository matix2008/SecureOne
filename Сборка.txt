1. В общем случае сборка решения SecureOne.sln (далее Решение) производится с помощью стандартной платформы для сборки приложений - MSBuild. Сборка Решения может производится в среде с установленной Visual Studio 2017+ (далее VS) или в среде без VS.

2. Установите MSBuil. Если у вас есть Visual Studio, то MSBuild уже установлен. В Visual Studio 2022 средства сборки устанавливаются в папку установки Visual Studio. Для обычной установки по умолчанию в Windows 10 файл MSBuild.exe находится в папке установки MSBuild\Current\Bin. В установщике Visual Studio перейдите в раздел Отдельные компоненты и найдите флажок для MSBuild. Он выбирается автоматически при выборе любой из других рабочих нагрузок для установки. Чтобы установить MSBuild в системе без Visual Studio, см. раздел "Средства сборки для Visual Studio 2022" на странице загрузок - https://visualstudio.microsoft.com/ru/downloads/. 

3. Создате в рабочей директории в которой вы собирайтесь производить сборку Решения каталог SecureOne и скопируйте в него исходные тексты решения.

4. Если вы ходитие осуществлять сборку Решения средствами IDE VS, то два раза кликните на файле SecureOne.sln. После загрузки IDE и Решения 
   а. выберите в верхнем меню: Сборка -> Диспетчер конфигураций;
   б. в открывшемся диалоге установите активную кофигурацию "Release" и нажмите кнопку "Закрыть";
   в. выберите в верхнем меню: Сборка -> Сборка решения (F6).
   г. Дожитесь окончания сборки Решения.

5. Если вы ходитие осуществлять сборку Решения из командной строки:
   а. запустить консоль командной строки
   б. запустите файл VSDevCmd.bat, находящийся в подпапке Common7/Tools каталога, в котором установлено решение Visual Studio.
   в. перейдите в созданный вами каталог SecureOne
   г. выполните команду MSBuild.exe -property:Configuration=Release

6. Собранные файлы находятся в каталоге SecureOne\SecureOne\bin\Release
